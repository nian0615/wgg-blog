# Node

## 同步、异步、阻塞、非阻塞
同步和异步关注的是消息通信机制，是针对IO而言
阻塞与非阻塞关注的是程序在等待调用结果时的状态，是针对线程而言
不能说“异步一定是非阻塞”，阻塞与非阻塞只是策略，异步用阻塞策略当然也可以，但是这样就发挥不出异步的优势了，没有意义，所以异步常常搭配非阻塞策略来使用。
